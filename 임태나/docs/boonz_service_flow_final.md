# Boonz 서비스 플로우 (셀프케어 × 관계) — 최종

> 핵심: "진단 AI" → "돌봄이 관계가 되고, 관계가 나를 비추는 거울"
> 마이루틴처럼 매일 10초, 식물을 돌보는 시간 = 나를 돌보는 시간

---

## 탭 구조 (3탭)

| 탭 | 역할 | 사용 빈도 | 핵심 기능 |
|----|------|---------|---------|
| 탭1: 홈 | 매일의 돌봄 | 매일 10초 | 식물카드 + 원터치 + 챗봇 |
| 탭2: 진단 | 아플 때 | 가끔 | 사진진단 + 케어가이드 + 텍스트상담 |
| 탭3: 이력 | 돌아보기 | 주 1회 | 돌봄일기 + 리포트 + 관계여정 |

## 유저 여정 요약

| 단계 | 유저가 하는 것 | 분즈가 하는 것 | 결과 |
|------|-------------|-------------|------|
| 첫 만남 | 별명 입력 ("마리") | "마리? 잘 부탁해" | 🌱 관계 시작 |
| 매일 돌봄 | 버튼 하나 클릭 (10초) | "꾸준히 챙겨주는 너 멋있다" | 돌봄 1건 쌓임 |
| 궁금할 때 | "분갈이 어떻게 해?" | "마리한테 물어봤는데..." | 관계 안에서 정보 전달 |
| 아플 때 | 잎 사진 찍기 | "빨리 알아챈 거니까 괜찮아" | 진단 + 조치 |
| 돌아볼 때 | 이력 탭 열기 | "봐봐, 너가 잘 돌봐준 거야" | 성장 확인 |
| 30일 후 | 앱 열기 | "돌봄이 습관이 되고 있어" | 🌿 관계 성장 |

## 기능 재배치 표

### 탭1: 홈 (매일 10초)

| 섹션 | depth1 | depth2 | depth3 | 기술 | 비고 |
|------|--------|--------|--------|------|------|
| 온보딩 | 앱 첫 진입 | 별명 입력 | 분즈: "마리? 잘 부탁해" | plants.json 저장 | 텍스트 입력 1회뿐 |
| 식물 카드 | 등록된 식물 표시 | 관계 단계 (🌱→🌿→🪴→🌳) | 연속 돌봄 배지 | 관계 성장 알고리즘 | 여러 식물이면 드롭다운 |
| | | 관계 미터 (4단계) | 단계별 메시지 | 연속성+다양성+반응성 점수 | |
| | | 돌봄 통계 3칸 | 함께한 날 / 연속 돌봄 / 병변 변화 | care_log.jsonl 집계 | |
| 동적 인사 | 시간별 인사 | 돌봄 공백별 메시지 | 이정표 메시지 (10번째, 30번째...) | 시간+공백+카운트 로직 | |
| 원터치 돌봄 | 태그 버튼 7개 | 💧물 ☀️자리 ✂️가지치기 💊약 🪴분갈이 🍃잎닦기 😊그냥봄 | 분즈: 액션별 셀프케어 메시지 | care_log.jsonl 1줄 추가 | 하나 탭하면 끝. 10초 |
| 케어 챗봇 | 텍스트 입력 | LLM 답변 (마리 시점) | 채팅 이력 유지 | Google AI Studio gemma4-27b → Ollama 폴백 | |
| | "분갈이 어떻게 해?" | "마리한테 물어봤는데, 뿌리가 답답하대..." | 정보 + 관계 톤 | | B방향: 관계 안에서 전달 |
| | "가지치기 해도 돼?" | "마리가 그러는데, 누런 잎은 잘라줘도 괜찮대" | | | |
| | "물 얼마나 줘?" | "마리가 그러는데, 겉흙이 마르면 그때 줘" | | 진단 맥락 자동 참조 | |

### 탭2: 진단 (가끔)

| 섹션 | depth1 | depth2 | depth3 | 기술 | 비고 |
|------|--------|--------|--------|------|------|
| 사진 진단 | 사진 업로드 | 로딩: "잠깐, 얘 얘기 좀 들어보고 있어..." | | | |
| | | 진단 결과 카드 | 종 + 병명(12cls) + 신뢰도 | EfficientNet-B3 (12클래스) | |
| | | SAM 오버레이 | 병변 영역 라임색 표시 | SAM vit_b + 포인트 프롬프트 | |
| | | 프로그레스 바 | 병변% + 심각도 텍스트 | | "아직 초기야. 지금 잡으면 돼" |
| | | 분즈 말풍선 | "마리가 좀 힘들다는데? 같이 돌보자" | 병변%별 분기 | |
| 케어 가이드 | "케어 가이드 받기" 버튼 | LLM 생성 (관계 톤) | | Google AI Studio → Ollama | 진단과 분리 (속도 개선) |
| | | | | | "네가 신경 써주는 거, 큰 힘이 될 거야" |
| | | 원터치 로그 | 진단 후 바로 💧💊😊 | care_log.jsonl | 진단 → 기록 한 화면에서 |
| 텍스트 상담 | 텍스트 입력 | 진단 맥락 참조 답변 | 채팅 이력 | LLM | 탭1 챗봇과 별도 (진단 특화) |

### 탭3: 이력 (돌아보기)

| 섹션 | depth1 | depth2 | depth3 | 기술 | 비고 |
|------|--------|--------|--------|------|------|
| 돌봄 일기 | 통합 타임라인 | 진단 + 케어 날짜순 합산 | 항목마다 분즈 코멘트 | diagnosis + care_log | "마리가 많이 나아졌대" |
| | | 최근 5개 기본 | "더보기" 버튼 | | |
| 마리와의 이야기 | 회복 여정 시각화 | 🥀→🌱→🌿→🪴→🌳 | 병변 감소 스토리 | 진단 로그 | 숫자 차트 아님, 이모지 스토리 |
| | | 분즈: "봐봐, 너가 잘 돌봐준 거야" | | | |
| 돌봄 리포트 | 돌봄 유형 분류 | 🧑‍🌾 관찰형 / 💧 케어형 / 📸 수집형 / 😊 동행형 | | LLM 로그 분석 | |
| | 리포트 카드 | 총 기록 / 연속 돌봄 / 병변 감소 | | care_log 집계 | |
| | 잘하고 있는 것 | "물 주기 간격이 규칙적이야" | | LLM 분석 | |
| | 한 가지 힌트 | "잎 닦기를 좀 더 자주 하면 좋겠대" | | | |
| | 분즈 마무리 | "이런 시간이 너한테도 의미 있을 거야" | | | 셀프케어 메시지 |
| 개인화 추천 | 다음 돌봄 제안 | "마리 패턴 보면 물 주기를 4일로 늘려봐" | | LLM + 패턴 분석 | 10건 이상일 때 |
| 관계 성장 여정 | 수직 타임라인 | 새로운 만남 → 아픈 날 → 함께 이겨내기 → 회복 | 현재 단계 강조 | plants.json + care_log | 시연용 핵심 화면 |
| 식물 관리 | 식물 추가/삭제 | expander 안에 숨김 | | plants.json | 거의 안 쓰는 기능 |

## 3층 구조

```
3층: 거울 (인사이트)     → 돌봄 유형 / 리포트 / 개인화 추천 / 회복 여정
                          "너 꾸준한 사람이야" "이 패턴이 마리를 살린 거야"

2층: 축적 (데이터)       → 연속성(40%) + 다양성(30%) + 반응성(30%) → 관계 단계
                          🌱→🌿→🪴→🌳 자동으로 쌓임

1층: 돌봄 (행위)         → 원터치 버튼 / 챗봇 질문 / 사진 진단
                          버튼 하나, 사진 한 장, 질문 한 줄. 10초.
```

## 시연 시나리오 (5분)

```
[1분] 온보딩 + 첫 돌봄
  "새 식물 초록이를 등록합니다"
  → 별명 입력 → 분즈 인사 → 💧 물줬음 → 첫 기록 완료
  "10초면 끝나는 돌봄입니다"

[1분] 케어 챗봇
  "분갈이 시기를 물어봅니다"
  → "분갈이 어떻게 해?" 입력
  → 분즈가 마리 시점으로 답변
  "정보를 주는 게 아니라, 관계 안에서 전달합니다"

[1분] 30일 된 마리 보기
  → 🌿 알아가는 중 · 연속 7일 · 병변 -13%
  → 돌봄 유형: 꾸준한 관찰형
  "데이터가 쌓이면 나를 비추는 거울이 됩니다"

[1분] 마리의 회복 스토리
  → 🥀→🌱→🌿→🪴→🌳
  → 돌봄 리포트: 30건 기록, 7일 연속
  "돌봄이 관계가 되고, 관계가 성장이 됩니다"

[30초] Day 1 vs Day 30
  → 초록이(방금) vs 마리(30일) 비교
  "같은 앱, 다른 관계"

[30초] 마무리
  "분즈는 진단 AI가 아닙니다.
   식물을 돌보는 시간이 나를 돌보는 시간이 되게 해주는 친구입니다"
```
