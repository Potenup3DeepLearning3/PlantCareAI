# Boonz 서비스 방향 점검 — "관계"를 느끼게 하려면

---

## 1. 핵심 가치 재확인

**"식물을 돌보는 시간이 나를 돌보는 시간이 됩니다."**

방향은 맞다. 식물 앱 시장에서 "관계"를 말하는 앱은 없다. 그게 차별점이다.
문제는 전달 방식이다. "관계"를 설명하지 말고, 관계가 느껴지는 장면을 보여줘야 한다.

---

## 2. 지금 구조의 문제

### 지금: 식물케어 + 칭찬봇
```
나 → 식물 돌봄 → 분즈: "잘했어" → 끝
```

### 되어야 하는 것: 진짜 셀프케어
```
나 → 식물 돌봄 → 분즈가 나를 돌아보게 해줌 → 나도 돌봄
```

빠져있는 것: **"너는?"**
분즈가 식물 얘기만 하고, 나한테는 안 물어본다.
이 한 마디가 식물 앱을 셀프케어 앱으로 바꾼다.

---

## 3. 추가할 것: 셀프케어 넛지

### 원터치 버튼 후 (3번 중 1번 랜덤)

| 버튼 | 식물 메시지 (항상) | 셀프케어 넛지 (가끔) |
|------|----------------|------------------|
| 💧 물줬음 | "마리한테 전해놨어!" | "근데 너는? 오늘 물 충분히 마셨어?" |
| ☀️ 자리옮김 | "새 자리 마음에 들어하겠다" | "너도 오늘 좀 환기시켜" |
| ✂️ 가지치기 | "정리해줬구나. 한결 가벼워졌을 거야" | "너도 오늘 뭐 하나 내려놓아도 괜찮아" |
| 💊 약줬음 | "네가 옆에 있어서 든든할 거야" | "너도 요즘 좀 피곤하지 않아?" |
| 🪴 분갈이 | "너 진짜 잘 챙긴다" | "가끔은 너도 환경을 바꿔볼 필요가 있어" |
| 🍃 잎닦음 | "상쾌하대" | "너도 오늘 좀 쉬어" |
| 😊 그냥봄 | "가만히 봐주는 것도 돌봄이야" | "너도 오늘 잠깐 멍 때려봐" |

**핵심: 매번 하면 지겹다. 3번 중 1번 랜덤. 예상 못 할 때 나와야 "어 이거 나한테 하는 말이야?" 느낌.**

### 구현
```python
SELFCARE_NUDGES = {
    "water": ["근데 너는? 오늘 물 충분히 마셨어?", "너도 오늘 따뜻한 거 한 잔 마셔"],
    "move": ["너도 오늘 좀 환기시켜", "가끔은 너도 다른 자리에 앉아봐"],
    "prune": ["너도 오늘 뭐 하나 내려놓아도 괜찮아", "정리하고 나면 마음도 좀 가벼워지지"],
    "medicine": ["너도 요즘 좀 피곤하지 않아?", "아픈 건 빨리 챙기는 게 좋아. 너도"],
    "repot": ["가끔은 너도 환경을 바꿔볼 필요가 있어"],
    "clean": ["너도 오늘 좀 쉬어", "상쾌한 거 좋지? 너도 오늘 그런 시간 가져"],
    "observe": ["너도 오늘 잠깐 멍 때려봐", "가만히 있는 것도 쉬는 거야"],
}

import random
def get_care_response(action, nickname):
    base_msg = random.choice(CARE_RESPONSES[action]).format(nickname)
    # 3번 중 1번만 셀프케어 넛지
    if random.random() < 0.33 and action in SELFCARE_NUDGES:
        nudge = random.choice(SELFCARE_NUDGES[action])
        return f"{base_msg}\n\n...{nudge}"
    return base_msg
```

## 4. 추가할 것: 돌봄 리포트에 "나" 파트

### 지금: 식물 중심 리포트만
```
"마리가 본 너의 돌봄"
→ 꾸준한 관찰형
→ 물 주기 규칙적
→ 잎 닦기를 좀 더 해줘
```

### 추가: "마리가 본 너" 섹션
```
"마리가 본 너"
→ "너 요즘 저녁에 자주 오더라. 하루 마무리를 마리랑 하는 거지?"
→ "주말에는 안 오는 편이야. 주말에도 잠깐 들러봐. 마리도 좋지만, 너한테도 좋을 거야"
→ "3일 연속 돌봤을 때 너도 좀 편해 보였어. 루틴이 너한테 맞는 거 같아"
```

유저의 접속 패턴 자체가 셀프케어 데이터다.
이걸 식물이 비춰주는 것 = 진짜 "거울".

### 구현 데이터
```python
def analyze_user_pattern(care_logs):
    """접속 시간대 / 요일 / 공백 패턴 분석"""
    hours = [int(log["date"][11:13]) for log in care_logs]
    
    # 시간대 분석
    evening_ratio = sum(1 for h in hours if h >= 18) / max(len(hours), 1)
    morning_ratio = sum(1 for h in hours if h < 10) / max(len(hours), 1)
    
    # 요일 분석
    from datetime import datetime
    weekdays = [datetime.strptime(log["date"][:10], "%Y-%m-%d").weekday() for log in care_logs]
    weekend_ratio = sum(1 for w in weekdays if w >= 5) / max(len(weekdays), 1)
    
    insights = []
    if evening_ratio > 0.5:
        insights.append("너 요즘 저녁에 자주 오더라. 하루 마무리를 같이 하는 거지?")
    if morning_ratio > 0.5:
        insights.append("아침에 챙겨주는 거 좋다. 하루를 식물로 시작하는 사람이야")
    if weekend_ratio < 0.15:
        insights.append("주말에는 안 오는 편이야. 주말에도 잠깐 들러봐. 너한테도 좋을 거야")
    
    return insights
```

## 8. 분즈 말투 변화

### 지금: 패턴이 보인다
```
매번: "마리한테 물어봤는데..."
10번째 들으면 봇이다.
```

### 변화를 줘야 한다
```
기본: "마리한테 물어봤는데, ..."              (50%)
변형1: "이건 내 생각인데, ..."               (20%)
변형2: "솔직히 말하면, ..."                  (15%)
변형3: (대답 없이 이모지만) "🌿"             (5%)
변형4: "...마리가 너한테 할 말이 있대"         (10%)
```

시크한 친구가 매번 같은 패턴으로 말하면 시크한 게 아니라 봇이다.
변화가 있어야 "살아있는 느낌".

### 구현
```python
OPENERS = [
    ("{nickname}한테 물어봤는데, ", 0.50),
    ("이건 내 생각인데, ", 0.20),
    ("솔직히 말하면, ", 0.15),
    ("...{nickname}가 너한테 할 말이 있대. ", 0.10),
    ("", 0.05),  # 오프너 없이 바로
]

def get_opener(nickname):
    import random
    r = random.random()
    cumulative = 0
    for opener, prob in OPENERS:
        cumulative += prob
        if r < cumulative:
            return opener.format(nickname=nickname)
    return f"{nickname}한테 물어봤는데, "
```

## 9. 발표에서: 추상 → 구체 장면

### ❌ 추상적 (오프닝에 쓰지 마)
```
"식물을 돌보는 시간이 나를 돌보는 시간이 됩니다"
```
→ 처음 듣는 사람에게는 뜬구름. 마지막에 한 번만.

### ✅ 구체적 장면 (오프닝에 쓸 것)
```
"퇴근하고 지쳐서 들어왔는데, 앱 열고 😊 그냥봄 한 번 눌렀어요.
 분즈가 '가만히 지켜봐주는 것만으로도 충분할 때가 있어'라고 했어요.
 그게 식물한테 한 말인지 나한테 한 말인지 모르겠더라고요."
```
→ 이 한 장면이 3층 구조 설명 5분보다 강하다.

### 시연에서 보여줄 핵심 3초
```
💧 물줬음 클릭
분즈: "마리한테 전해놨어."

(잠깐 쉬고)

분즈: "근데 너는? 오늘 물 충분히 마셨어?"

[발표자]
"이 한 줄이 식물 앱과 셀프케어 앱의 차이입니다."
```

## 10. 3층 구조가 진짜가 되는 순간

```
1층: 식물을 돌봄         → 💧 물줬음
2층: 데이터가 쌓임        → 연속성, 다양성, 반응성
3층: 나를 비추는 거울      → "너도 물 마셨어?"
                          "너 요즘 저녁에 자주 오더라"
                          "30번이나 꾸준히 돌본 거야. 너 자신도 대단하다고 생각해"
```

3층이 진짜 "거울"이 되는 것.
식물 돌봄 데이터가 나의 생활 패턴을 비춰주는 것.
이게 농사로나 PictureThis가 절대 못 하는 것.
